## Early Session ‚Äî Core Transaction Flow Bugs

### 1. Amount showing as 0ƒë (SePay payload parsing)
- **Problem:** SePay sends data nested under a `"data"` key, but bot was reading from top-level ‚Äî all fields came back empty/0
- **Fix:** Added data unwrapping: `data = payload.get("data") if "data" in payload else payload`
- **Also fixed:** Added fallbacks for all known SePay field names: `transferAmount` ‚Üí `transfer_amount` ‚Üí `amount`
- **Debug added:** `print("DEBUG sepay raw payload:", json.dumps(payload))` to diagnose field names
- **File:** `handlers/sepay.py`

### 2. Wrong row number ‚Äî transactions written to row ~1001
- **Problem:** `append_transaction()` returned the correct row number, but `sepay.py` discarded it and called `get_last_row()` separately ‚Äî which counted ALL rows including empty ones at the bottom of the sheet ‚Üí returned ~1001
- **Fix:** Used return value directly: `row_num = sh.append_transaction(...)`
- **File:** `handlers/sepay.py`, `sheets.py`

### 3. Spending counter staying 0% after categorizing
- **Root cause:** Directly caused by bug #2 ‚Äî wrong `row_num` meant `finalize_transaction()` wrote category/confirmed flags to the wrong row (row ~1001) ‚Üí `get_bucket_status()` couldn't find confirmed transactions ‚Üí `spent = 0` ‚Üí 0%
- **Fixed by:** Fixing bug #2

### 4. Column-shifting bug in Google Sheets
- **Problem:** `gspread`'s `ws.append_row()` detected the sheet's table starting at column F (legacy Google Apps Script structure) and shifted all data 5 columns right ‚Äî date ended up in col F instead of col B
- **Fix:** Replaced `append_row()` with explicit range write: `ws.update(f"A{n}:O{n}", [row_data])`
- **File:** `sheets.py` ‚Üí `append_transaction()`

### 5. Duplicate rows ‚Äî SePay native Sheets integration conflict
- **Problem:** SePay had its own built-in Google Sheets integration writing rows directly to the sheet AND the bot was also writing ‚Äî bot checked `ref_code` in sheet to dedup, but SePay's native row was already there (using `content` field, amount=0), so bot thought it was a duplicate and skipped sending Telegram message
- **Fix:** Switched to in-memory dedup (`_processed_refs` dict with 5-min TTL) ‚Äî no longer checks the sheet
- **Action required:** Disabled SePay's native Google Sheets integration
- **File:** `sheets.py` ‚Üí `tx_exists()`

### 6. Budget allocation not saving / overwriting wrong rows
- **Problem:** `write_budget_row()` only appended new rows and skipped existing ones silently ‚Äî `find_budget_row()` returned `True` (row exists) but then nothing happened
- **Fix:** Rewrote to find existing row by index and use `ws.update()` on cols C:F to update in-place
- **File:** `sheets.py` ‚Üí `write_budget_row()`

### 7. 0% displayed on small transactions
- **Problem:** Python uses banker's rounding ‚Äî `round(0.5) = 0` ‚Äî so small amounts like 14,000ƒë / 100,000ƒë cap = 14% showed as 0%
- **Fix:** Added `calc_pct()` helper: uses `int()` truncation (not `round()`), returns minimum 1% when any spending exists
- **File:** `sheets.py` ‚Üí `calc_pct()`

### 8. Weekly summary showing no transactions
- **Problem:** Stored datetimes in sheet were naive (no timezone info) ‚Äî when loaded and compared, Python treated them as UTC, converted to VNT added +7h ‚Üí transactions appeared to be in the future
- **Fix:** Used `tz.localize(d)` instead of `pytz.utc.localize(d)` when loading stored dates
- **File:** `handlers/reports.py`

### 9. `/allocation` not finding previous month data to copy
- **Problem:** Previous month (March 2026) had no data in Budget Config ‚Üí allocation flow found nothing to copy ‚Üí showed empty
- **Fix:** Updated `start_monthly_allocation()` to try current month first, then fall back to previous month as copy source
- **File:** `handlers/allocation.py`

### 10. State lost between parent ‚Üí sub-category picker steps
- **Problem:** `handle_parent_selected()` set a fresh state dict, overwriting the `amount` field saved from the webhook step ‚Üí `_finalize()` saw `amount = 0`
- **Fix:** Spread previous state before updating: `sh.set_state(CHAT_ID, {**prev_state, "step": "await_sub", ...})`
- **File:** `handlers/transaction.py`
